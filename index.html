System configuration

Lab Setup

Complete all four setup steps before the session starts. Estimated time: 20–30 minutes.

1

Google Sheets — Equipment Inventory

Download the pre-built inventory workbook and save it as a native Google Sheet in your Drive.

  • Go to drive.google.com → New → File upload and select the downloaded file
  • Right-click the uploaded file → Open with → Google Sheets
  • Confirm all tabs are present: Stock, Reservations, Entitlements, Employee Seed Data
2

Salesforce — Employee Contacts

Step A — Create the Equipment Role custom field

  • Gear icon → Setup → Object Manager → Contact → Fields & Relationships → New
  • Select Picklist → Next
  • Field Label: Equipment Role — API name auto-fills as Equipment_Role__c
  • Paste the values below, then click Next → Next → Save
Software Engineer Senior Engineer Product Manager Designer Sales IT Admin Manager / Director Intern

Step B — Create two Contact records

  • Contact 1 (you — the employee): your name · your email · Equipment Role = Software Engineer · Department = Engineering
  • Contact 2 (your manager alias): your name + "(Manager)" · you+manager@gmail.com · Equipment Role = Manager / Director
  • On Contact 1, set Reports To → Contact 2
Tip: Gmail aliases work seamlessly — you+manager@gmail.com delivers to your inbox but appears as a distinct address to Salesforce and the Gmail send action in your recipe.
3

Jira Service Management — IT Helpdesk Space

  • Open your Atlassian site → select Jira Service Management
  • Click Create project → choose IT Service Management template → Next
  • Space name: IT Helpdesk · Key: ITHD → Create
4

Workato — Create Connections

In your Workato workspace go to Connections → New connection and authenticate each app. Verify all four show green Connected status before starting.

MCP Server

Tool Reference

Five tools exposed by the ATLAS MCP server, each wired to a live business application.

ATLAS
Automated Technology Logistics & Allocation System
Handles IT equipment requests — checks eligibility, inventory, raises tickets and sends notifications.
5tools
check_inventoryGoogle Sheets
Returns available IT equipment from the live inventory sheet. Filters out out-of-stock items automatically. Call this before any provisioning request to confirm items are available. Optionally filter by equipment category to narrow results.
Inputs
categorystringoptional
Filter by: Laptop, Monitor, Keyboard, Mouse, Audio, Accessory. Leave empty to return all available items.
Output
itemsarray
List of available inventory items. Only items with qty > 0 included.
items[].item_idstring
Unique identifier, e.g. ITEM-001. Pass this to reserve_item.
items[].item_namestring
Display name, e.g. MacBook Pro 14" M3.
items[].categorystring
Equipment category.
items[].available_qtyinteger
Current available stock quantity.
get_employee_profileSalesforce
Looks up an employee in Salesforce and returns their profile including role, department, manager contact, and full equipment entitlement tier. Always call this before reserving any item — the entitlement object determines whether the requested equipment is eligible for this employee's role.
Inputs
employee_emailstringrequired
Corporate email of the employee. Used to look up their Contact record in Salesforce.
Output
employee_idstring
Salesforce Contact ID. Pass to reserve_item.
full_namestring
Employee's full name. Pass to reserve_item and send_approval_request.
rolestring
Equipment Role value, e.g. Software Engineer.
departmentstring
Employee's department.
manager_emailstring
Manager's email. Pass to send_approval_request.
entitlementobject
Full entitlement row — laptop tier, monitor limit, keyboard, mouse, audio, accessories eligibility.
reserve_itemGoogle Sheets
Reserves one or more items from inventory on behalf of an employee. Decrements Available Qty and increments Reserved Qty in the Stock tab, then appends one row per item to the Reservations tab. Only call after confirming eligibility via get_employee_profile and availability via check_inventory.
Inputs
employee_emailstringrequired
Email of the employee requesting the items.
employee_idstringrequired
Salesforce Contact ID from get_employee_profile. Logged for traceability.
employee_full_namestringrequired
Full name from get_employee_profile. Written to Reservations tab.
itemsarrayrequired
List of items to reserve. Each entry requires item_id and quantity.
items[].item_idstringrequired
Item identifier from check_inventory, e.g. ITEM-001.
items[].quantityintegerrequired
Number of units to reserve. Minimum 1.
Output
reservation_idstring
Unique reservation ID, e.g. RES-20250320143022. Pass to create_provisioning_ticket and send_approval_request.
statusstring
Always returns "Pending" on successful reservation.
create_provisioning_ticketJira JSM
Creates an Equipment Request issue in the ITHD Jira Service Management space. Captures the employee, items requested as a human-readable string, reservation ID, and priority. Call this after reserve_item succeeds. Returns the ticket key and URL for use in the approval email.
Inputs
employee_emailstringrequired
Included in the ticket description for IT admin reference.
reservation_idstringrequired
From reserve_item — included for cross-system traceability.
itemsstringrequired
Human-readable list, e.g. "MacBook Pro 14" M3 ×1, Magic Keyboard ×1".
prioritystringoptional
Low, Medium, or High. Defaults to Medium.
Output
ticket_keystring
Jira issue key, e.g. ITHD-42.
ticket_urlstring
Direct URL to the Jira issue. Pass to send_approval_request.
reservation_idstring
Echo of the input — returned so the agent doesn't need to track state between calls.
statusstring
Initial ticket status, always "Open" on creation.
send_approval_requestGmail
Sends a formatted approval email to the employee's manager with full provisioning details — employee name, items requested, Jira ticket link, and reservation ID. Call this after create_provisioning_ticket succeeds. This is the human-in-the-loop checkpoint in the provisioning flow.
Inputs
manager_emailstringrequired
From get_employee_profile output field manager_email.
employee_namestringrequired
Full name of the requesting employee — used in the email subject and body.
itemsstringrequired
Human-readable items list — same string passed to create_provisioning_ticket.
ticket_urlstringrequired
From create_provisioning_ticket — so the manager can review and approve.
reservation_idstringrequired
From create_provisioning_ticket output — included for cross-system reference.
Output
statusstring
Returns "Approval email sent" on success.
sent_tostring
The manager email address the approval request was delivered to.
